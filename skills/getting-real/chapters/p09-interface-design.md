# Interface Design

## When to Load This

Load this when the question is about screens, flows, defaults, empty states,
errors, or how the product should feel in use.

## Core Idea

Design the actual experience first. Start from the core use case, handle the empty and broken states,
and use language as part of the interface instead of decoration.

## Chapter Routing

- **Ch 46 Interface First** -> Start from what users will actually see and do.
- **Ch 47 Epicenter Design** -> Find the product's center of gravity and design around it.
- **Ch 48 Three State Solution** -> Cover regular use, blank state, and error state.
- **Ch 49 The Blank Slate** -> Empty states need direction, not awkward silence.
- **Ch 50 Get Defensive** -> Design for mistakes, misuse, and failure paths.
- **Ch 51 Context Over Consistency** -> Consistency is not the goal if local clarity is better.
- **Ch 52 Copywriting is Interface Design** -> Labels and helper text are product behavior.
- **Ch 53 One Interface** -> Avoid fragmented experiences or special-case UI sprawl.

## Practical Reads

- This part is excellent for product reviews.
- If UX is "technically fine" but still feels confusing, read Ch 52 immediately.
- The book cares about usefulness more than design-system dogma.
