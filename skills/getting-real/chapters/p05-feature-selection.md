# Feature Selection

## When to Load This

Load this when a feature is being proposed, debated, defended, or smuggled in.

## What This Part Covers

- Chapter 21: Half, Not Half-Assed
- Chapter 22: It Just Doesn't Matter
- Chapter 23: Start With No
- Chapter 24: Hidden Costs
- Chapter 25: Can You Handle It?
- Chapter 26: Human Solutions
- Chapter 27: Forget Feature Requests
- Chapter 28: Hold the Mayo

## Core Idea

Most feature decisions should begin with skepticism. The right question is not "can we build it?"
but "should this exist at all, and if so, what is the smallest honest version?"

## Chapter Routing

- **Ch 21 Half, Not Half-Assed** -> Cut to a narrower version that is still coherent and useful.
- **Ch 22 It Just Doesn't Matter** -> The debated thing is low leverage and should be ignored.
- **Ch 23 Start With No** -> Default filter for any new feature pitch.
- **Ch 24 Hidden Costs** -> Reminder that every feature creates support, design, testing, and mental overhead.
- **Ch 25 Can You Handle It?** -> Ask whether the team can carry this feature for the long term.
- **Ch 26 Human Solutions** -> Replace software complexity with manual handling where sane.
- **Ch 27 Forget Feature Requests** -> Do not let user requests become a raw roadmap feed.
- **Ch 28 Hold the Mayo** -> Strip optional extras and keep the product clean.

## Practical Reads

- This is the first file to load for PM debates.
- If a feature is justified by edge cases, this part usually wants to kill it.
- If you must keep a feature, shrink it until it stops poisoning the rest of the product.
