# Introduction

## When to Load This

Load this file when the question is about the book's philosophy, whether Getting Real applies,
or why the authors push such a blunt style.

## What This Part Covers

- Chapter 1: What is Getting Real
- Chapter 2: About Basecamp
- Chapter 3: Caveats, disclaimers, and other preemptive strikes

## Core Idea

Getting Real is a reaction against overbuilt products and over-theorized process. Build the real thing sooner,
use constraints as leverage, and accept that the book speaks in bold strokes on purpose.

## Chapter Routing

- **Ch 1** -> Load when someone needs the core definition: less, more real screens, faster iteration.
- **Ch 2** -> Load when context on Basecamp's worldview helps explain the advice.
- **Ch 3** -> Load when someone objects that these ideas are too absolute or only work for small teams.

## Agent Notes

- The tone matters. The book is intentionally provocative to force action.
- Do not dilute the philosophy into "balance is important" mush.
- If the team is drowning in representation instead of product, this part is the reset.
