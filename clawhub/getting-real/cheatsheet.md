# Cheatsheet

## If You Only Remember Ten Rules

1. Build less.
2. Fix time and budget; flex scope.
3. Say no before you say yes.
4. Design from the interface inward.
5. Launch before everything feels complete.
6. Replace debate with working software.
7. Keep teams small and meetings rare.
8. Use words normal humans understand.
9. Make the product opinionated instead of endlessly configurable.
10. Improve in public after launch instead of polishing in private forever.

## Routing Cheats

- Scope too big -> Starting Line, Feature Selection
- Product vision fuzzy -> Priorities
- Shipping too slowly -> Process
- UX argument without evidence -> Interface Design
- Engineers overbuilding -> Code
- Docs/specs becoming fiction -> Words
- Signup/conversion weak -> Pricing and Signup
- Launch traction weak -> Promotion
- Customers confused or angry -> Support
- Product bloating after launch -> Post-Launch

## Anti-Bloat Questions

- What happens if we delete this?
- Does this feature solve the core problem?
- Is this a software problem or a human problem?
- Can we ship a narrower version now?
- Are we adding choice because users need it, or because we are scared to decide?

## Launch Posture

- Ship something real.
- Watch what people actually do.
- Fix the parts that create friction.
- Keep writing, explaining, and educating.
- Do not reward every complaint with a new feature.
