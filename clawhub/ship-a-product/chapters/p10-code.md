# Code

## When to Load This

Load this when engineering decisions are making the product heavier, less understandable,
or less pleasant to build.

## What This Part Covers

- Chapter 54: Less Software
- Chapter 55: Optimize for Happiness
- Chapter 56: Code Speaks
- Chapter 57: Manage Debt
- Chapter 58: Open Doors

## Core Idea

The codebase should serve product speed and team sanity. Write less of it, keep it readable, and manage debt deliberately instead of mythologizing architecture.

## Chapter Routing

- **Ch 54 Less Software** -> Solve problems with less code when possible.
- **Ch 55 Optimize for Happiness** -> Developer joy matters because miserable teams build worse products.
- **Ch 56 Code Speaks** -> Prefer code truth over documentation theater.
- **Ch 57 Manage Debt** -> Some debt is fine if it is visible and controlled.
- **Ch 58 Open Doors** -> Build in ways that keep future options available.

## Practical Reads

- This part is a good antidote to engineering vanity.
- The book is not anti-quality; it is anti-bulk and anti-ceremony.
- If "best practice" is slowing the team without improving the user experience, question it.
